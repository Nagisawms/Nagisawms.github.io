JavaWeb学习

发表于 2019-11-02 分类于 Java进阶

JavaWeb知识总结

response:响应对象

1）提供的方法：
void addCookie( Cookie cookie ) 服务端向客户端增加cookie对象

void sendRedirect(String location ) throws IOException; 页面跳转的一种方式（重定向）

void setContetType(String type) 置服务端响应的编码（设置服务端的contentType类型）

2）转发与重定向的区别

请求转发：

地址栏:不变 

需要保留第一次请求时的数据

请求次数为一次

跳转发生的位置位于服务端.

重定向：

地址栏:改变

不需要保留第一次请求时的数据

请求次数为两次

跳转发生的位置位于客户端发出的第二次跳转.

3)下面用一个形象的例子来解释：

转发：

张三（客户端）     ->    【 服务窗口 A （服务端 ）    ->  服务窗口B  】

重定向：

张三（客户端）       ->     服务窗口 A （服务端 ） ->去找B

张三（客户端）    ->     服务窗口 B （服务端 ） ->结束

Cookie对象

　　Cookie（客户端，不是内置对象）:Cookie是由 服务端生成的 ，再发送给客户端保存。
相当于 本地缓存的作用： 客户端->服务端
作用：提高访问服务端的效率，但是安全性较差。

Visual Basic 文件流的操作

　　我最近参与的是日本一家大型证券公司的内部系统升级项目，该系统据我了解是2002年左右用微软vb6.0开发的，我们项目组要做的是用微软最新的.NETFreamwork4.7.2框架对该系统做一次全面的升级（可能马上又要4.8了），这个系统非常依赖各种ini配置文件，在各个页面的spread中的数据显示依靠处理程序直接读取csv文件中的各种业务数据，所以该系统对于文件流的读取就显得尤为重要了。

下面我们正式开始学习VB.NET对文件流的处理吧

文件I/O流模型

　　VB.NET用流（Sream）表示数据的传输操作，将数据从内存传输到某个载体或设备中，称为 输出流,反之，将数据从某个载体或设备传输到内存中，称为 输入流，更进一步推广流的概念，可以把与数据传输有关的事物称为 流。

　　VB.NET对文件的操作就是利用流来完成的。流的输入/输出是通过 System.IO模型来实现的，该模型提供了大量的类，可以将数据从流中读出或将流中的数据写入文件。
